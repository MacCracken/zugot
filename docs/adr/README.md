# Architecture Decision Records

Each file captures one decision: what was chosen, why, what was considered
instead, and what happens if we change our minds later.

Format: one markdown file per decision, named `NNNN-short-title.md`.

Status values:
- **proposed** — under discussion
- **accepted** — decided, implemented
- **deprecated** — superseded by a newer ADR (link to it)
- **rejected** — considered, decided against; kept for the reasoning
