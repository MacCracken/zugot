## ⚠️ Fixed this pass

### gnupg 2.4.9 → **2.5.18** — CVE-2026-24881 (CVSS 9.8 CRITICAL)

**Issue:** Stack-based buffer overflow in `gpg-agent` during `PKDECRYPT --kem=CMS`
handling. A crafted CMS (S/MIME) `EnvelopedData` message with an oversized
wrapped session key can trigger remote code execution or DoS. Disclosed
2026-01-27.

**Fix version:** 2.5.17 (upstream); 2.5.18 is the current head of the 2.5 line.

**Why migrate to 2.5:** The 2.4.x branch is EOL on 2026-06-30, and upstream did
**not** backport the fix. 2.5.x is the LTS-in-waiting track. Migration tested
with sigil signature verification on this host.

**Recipe change:** `base/gnupg.cyml` — version `2.4.9` → `2.5.18`, URL updated,
SHA256 verified from fresh tarball download (`0dbd64e0...`). Header comment
documents the CVE rationale.

**Sources:**
- [CVE-2026-24881 NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-24881)
- [Red Hat Bugzilla 2433480](https://bugzilla.redhat.com/show_bug.cgi?id=2433480)
- [SentinelOne advisory](https://www.sentinelone.com/vulnerability-database/cve-2026-24881/)
- [SUSE CVE page](https://www.suse.com/security/cve/CVE-2026-24881.html)

## 🟡 No upstream fix yet — documented, monitored

### sudo 1.9.17p2 — CVE-2026-35535 (CVSS 7.4)

**Issue:** `setuid`/`setgid`/`setgroups` call failure during privilege drop
before running the mailer is treated as non-fatal, enabling local privilege
escalation. CWE-271. Disclosed 2026-04-02.

**Fix status:** Commit `3e474c2` exists upstream but NO tagged release yet.
Latest tag on sudo.ws/dist is **1.9.17p2** (our current version).

**Action:** Monitor for `1.9.17p3`. If a release build is cut before upstream
tags, apply commit 3e474c2 as a patch.

- [CVE-2026-35535 NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-35535)
- [Sudo security advisories](https://www.sudo.ws/security/advisories/)

### glibc 2.43 — CVE-2026-4046 (CVSS 7.5)

**Issue:** `iconv()` assertion crash when converting from IBM1390 or IBM1399
character sets. Network-reachable DoS. Disclosed 2026-03-30.

**Fix status:** No upstream glibc 2.44 released yet (latest on ftp.gnu.org/gnu/glibc
is **2.43**). Upstream mitigation: remove IBM1390/IBM1399 from the charset
list in the local `gconv-modules` file at install time.

**Action:** Apply install-time charset-removal mitigation OR wait for 2.44.

- [CVE-2026-4046 NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-4046)

### libtiff 4.7.1 — CVE-2025-61144 (CVSS 7.3) + CVE-2025-61143

**Issue #1:** Stack overflow in `readSeparateStripsIntoBuffer()` — unpatched
at 4.7.1.

**Issue #2:** NULL-deref in `tif_open.c` (DoS) — also unpatched at 4.7.1.

**Fix status:** Upstream has not cut a 4.7.2 release. Our version is latest.

**Action:** Monitor for 4.7.2 or later.

- [CVE-2025-61144 GHSA-m6xw-mq4p-x7xv](https://github.com/advisories/GHSA-m6xw-mq4p-x7xv)
- [CVE-2025-61143](https://www.cve.news/cve-2025-61143/)

### binutils 2.46.0 — CVE-2026-4647 + CVE-2025-1147/1148/11839

**Issue:** BFD XCOFF out-of-bounds read (DoS + info leak) plus multiple older
bugs in `nm`, `ld`, and `prdbg.c`.

**Fix status:** Per-CVE patch status unclear; Red Hat has advisories
(RHSA-2026:0108, RHSA-2026:2627) with backported patches for RHEL. No upstream
2.46.1 tagged.

**Action:** Consider applying RHEL-style backport patches OR wait for
upstream 2.46.1. Flagged for next release-build decision.

- [RHSA-2026:0108](https://access.redhat.com/errata/RHSA-2026:0108)

### gstreamer 1.28.2 + plugins-base/good — CVE-2026-5056 (CVSS 7.8)

**Issue:** Per BSI advisory 2026-0023 (2026-04-07), gst-plugins-base/good/bad
≤1.28.2 have a local-vector vulnerability in media parsing.

**Fix status:** Fix pending in 1.28.3 — not yet released upstream.

**Action:** Monitor for 1.28.3. Low urgency (local vector only).

- [GStreamer security page](https://gstreamer.freedesktop.org/security/)

### libarchive 3.8.7 — CVE-2026-4111

**Issue:** RAR5 decompression infinite-loop DoS via `archive_read_data()`.
Disclosed 2026 Q2, medium severity.

**Fix status:** Not yet patched upstream. Our 3.8.7 is latest.

**Action:** Monitor for 3.8.8+.

- [Debian security tracker — libarchive](https://security-tracker.debian.org/tracker/source-package/libarchive)

### libxslt 1.1.45 — CVE-2025-11731 + others

**Issue:** Type confusion in `exsltFuncResultComp()` (EXSLT `func:result`),
DoS. Plus three older CVEs (CVE-2025-7424, 7425, 10911) with unclear patch
status in 1.1.45.

**Fix status:** Project under-maintained; no 1.1.46+ tagged.

**Action:** Flag for strategic review — is libxslt a dependency we can shed?
Most zugot consumers that need XSLT could be migrated to `xsltproc` (from
libxslt), but the library itself has maintenance-gap risk. Low urgency, but
note for future.

- [libxslt CVEs](https://cve.threatint.eu/CVE/CVE-2025-7424)

## CI / tooling

The zugot validator (`scripts/validate_recipes.py`) does not currently check
CVE status — it only checks recipe validity. A CVE-awareness CI step would
need to cross-reference pinned versions against NVD feed data. Noted in
`docs/development/roadmap.md` P3 for future work.
